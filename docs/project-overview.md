# Rill 项目介绍

## 项目定位

Rill 是一套从数据库内核到产品界面的完整作品集。  
它的目标不是只展示 SQL 结果，而是把一条真实路径讲清楚：

1. SQL 如何被编译与执行
2. 内核能力如何被服务端封装
3. 客户端与 Web 工作台如何把能力可视化

## 核心组成

- `rill-core`：数据库内核，承载编译、执行、catalog、存储、事务与恢复等能力
- `rill-server`：服务端协议壳，提供原生 TCP 与 MySQL 协议兼容入口
- `rill-client`：本地 CLI / GUI 客户端
- `rill-app-web`：Spring Boot Web 演示后端，连接认证、工作台业务和内核执行
- `web`：前端控制台与文档门户
- `rill-launcher`：`log/data` 等开发期工具入口

## 关键体验入口

- 游客：默认数据库、最小权限体验
- 用户：个人数据库 + 工作台会话
- 管理员：用户与数据库运维能力

## 一条主执行链路

1. 在 Web 工作台或 CLI 输入 SQL
2. 进入 `rill-app-web` 的会话与权限校验
3. 由查询服务进入 `rill-core` 执行
4. 返回结果、耗时与 trace
5. 在 UI 中呈现结果面板、历史、场景和导出任务

## 当前工程能力

- SQL 工作台执行与结果回显
- 执行链路 trace 展示
- SQL 片段管理
- 场景脚本保存与回放
- 导出任务创建、执行与下载
- 多数据库访问控制（含游客/用户/管理员边界）

## 发布形态

- Windows 安装包
- Linux 归档
- macOS 归档
- 纯 Web JAR
- 带 UI 的 Web JAR

## 适用场景

- 数据库课程或答辩展示
- 内核能力演示与调试
- 协议兼容实验（MySQL 客户端接入）
- 从底层到前端的一体化工程样例
