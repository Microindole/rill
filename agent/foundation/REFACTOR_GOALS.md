# 重构目标

## 总目标

把 `rill` 从“技术野心很强但工程形态粗糙的实训项目”，重构成“适合 Java 面试展示的系统型项目”。

## 保留项

必须保留的核心竞争力：

- 自研 SQL 处理链路
- 自研存储和索引能力
- 自研事务与恢复基础
- 系统型项目的技术辨识度

## 改进项

必须重点改进：

- 启动与运行方式
- 架构边界
- 模块职责
- 测试体系与覆盖率可见性
- 文档质量
- 面试表达能力

## 功能扩展原则

后续新增功能必须服务于项目定位，优先选择：

- 能强化数据库或数据平台特征的功能
- 能体现 Java 后端工程能力的功能
- 能被清晰解释的功能

避免新增：

- 与核心定位无关的表层功能
- 只追求数量、不追求质量的功能
- 为了“像大厂项目”而生拼硬凑的功能
