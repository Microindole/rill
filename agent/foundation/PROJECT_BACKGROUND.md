# 项目背景

## 背景

`rill` 最初是一个实训项目，但它的技术内核比普通 Java 练手项目更有深度。项目已经具备数据库系统相关的关键部件，因此具有继续重构和增强的价值。

## 为什么保留它

相比常见的 toy CRUD 项目，这个项目已经具备以下辨识度：

- SQL 编译链路
- 存储引擎基础设施
- B+Tree 索引
- 事务、日志、恢复
- 多种运行入口

## 为什么要重构

当前项目虽然技术含量不低，但仍带有明显的实训项目痕迹：

- 入口分散
- 运行方式不统一
- 演示代码和正式运行路径混在一起
- 文档和维护体系不足
- 测试体系在多轮重构后严重老化，必须按当前架构重新建立

重构目标不是把它变成普通业务模板，而是把它打磨成一个更成熟、更可信、更适合面试展示的系统型 Java 项目。
